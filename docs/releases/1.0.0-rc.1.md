# Hooka `1.0.0-rc.1`

`1.0.0-rc.1` is the first release candidate for Hooka as a generic, internet-exposed automation control plane.

Hooka's supported operating model for this release is:

- single node
- self-hosted deployment
- SQLite-backed queue and state store

## Included in this RC

- retry with exponential backoff
- dead-letter queue support via `dead-lettered`
- stuck run recovery based on worker lease expiry
- target-backed webhook execution with policy validation and artifact readiness checks
- admin bearer token protection for read/admin APIs
- in-memory rate limiting for `/api/*`
- SSE live updates for run events and worker heartbeats
- target APIs, CLI commands, and admin UI panels

## Minimum env for public deployment

- `HOOKA_WEBHOOK_SECRET`
- `HOOKA_ADMIN_TOKEN`
- `HOOKA_TARGETS_PATH`

For shared-volume Pages deploys, the worker also needs:

- `CLOUDFLARE_API_TOKEN`
- `CLOUDFLARE_ACCOUNT_ID`

## Operational notes

- Retryable failures are requeued with backoff until `maxAttempts` is exhausted.
- Retryable failures that exceed `maxAttempts` become `dead-lettered`.
- Validation, policy, and auth failures are terminal and non-retryable.
- Targets are file-backed by default at `.hooka/targets.json`.
- Existing generic and compatibility webhook contracts remain supported.
