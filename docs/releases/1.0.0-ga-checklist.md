# Hooka `1.0.0` GA Checklist

This checklist is for the final `1.0.0` cut and its operator-facing verification pass.

## Final smoke checklist

- `bun run check`
- `bun run lint`
- `bun run format:check`
- `bun test --timeout 30000`
- `bun run build`
- `bun run test:dev-ui:smoke`
- `bun run test:e2e:docker`
- `docker compose config`
- local `hooka dev`
- local generic webhook success path
- local target-based webhook success path
- retryable failure to dead-letter path
- protected API access with missing and valid `HOOKA_ADMIN_TOKEN`

## GHCR verification

- push the final `v1.0.0` tag
- confirm immutable tags publish successfully:
  - `ghcr.io/imjlk/hooka:1.0.0-webhook-server`
  - `ghcr.io/imjlk/hooka:1.0.0-cf-pages`
  - `ghcr.io/imjlk/hooka:1.0.0-cf-cache`
  - `ghcr.io/imjlk/hooka:1.0.0-wp-ops`
  - `ghcr.io/imjlk/hooka:1.0.0-wp-wrangler`
- verify public pull for at least:
  - `docker pull ghcr.io/imjlk/hooka:1.0.0-webhook-server`
  - `docker pull ghcr.io/imjlk/hooka:1.0.0-cf-pages`

## Coolify verification

- deploy `hookaserver` + `hookaworker`
- verify shared `/data` and `/shared-source` mounts
- verify `HOOKA_TRUST_PROXY=true` behind the public Coolify proxy
- verify admin UI loads with `HOOKA_ADMIN_TOKEN`
- verify `hooka audit list` returns remote audit events against the Coolify URL

## Final consistency checks

- confirm operator docs and release notes all match the final `1.0.0` tag
- confirm no deferred item in docs is already implemented
- confirm target scaffold UX and audit visibility are sufficient for GA

## Explicitly deferred after GA

- planned preset promotion
- richer target authoring beyond scaffold + JSON editor
- token rotation and stronger proxy-chain controls
- multi-node/Postgres support
